# Local Image Manager

Obsidian 插件：粘贴或拖拽图片时，自动保存到本地，按笔记路径存放，并按标题层级自动命名（如 `1.2.3-1.png`）。

**纯本地存储，无需网络，无任何第三方服务依赖。**


---

## ✨ 功能特性

- 📋 **粘贴即保存**：从剪贴板粘贴图片（截图、复制图片文件）时自动处理。
- 🖱️ **拖拽也支持**：从系统文件管理器或 Obsidian 内部拖拽图片到编辑器，同样自动处理。
- 🗂️ **按笔记路径存放**：图片保存在 `基础文件夹/笔记目录/笔记名/` 下，保持与笔记的关联。
- 🔢 **智能命名**：根据光标所在的标题层级生成带序号的名称（如 `1.2.3-1.png`），类似 Word 多级列表。
- 🔄 **自动计数**：每个标题层级下的图片序号从 1 开始递增，重启 Obsidian 后依然保持。
- ⚠️ **友好提醒**：当光标不在任何标题下时，会提示建议，同时使用 `笔记名-时间戳` 作为后备命名。
- 🔗 **链接格式切换**：支持 Wiki（`![[路径]]`）和 Markdown（`![]()`）两种链接格式，一键转换。
- 🔧 **重新整理序号**：一键修正所有图片的层级前缀和序号，使每个层级编号从 1 开始连续。
- 🖱️ **右键菜单（编辑器链接）**：在编辑器（源码模式）中，将光标放在图片链接（`![[...]]` 或 `![]()`）上右键，快速执行“整理序号”或“删除”。
- 🗑️ **安全删除**：删除图片时自动移至系统回收站，并移除笔记中所有指向该文件的链接。
- 📝 **无缝插入**：自动在编辑器中插入相对路径链接，无需手动调整。

## 📦 安装

### 手动安装
1. 在 Obsidian 仓库的 `.obsidian/plugins/` 目录下新建文件夹 `local-image-manager`。
2. 下载 [`main.js`](main.js) 和 [`manifest.json`](manifest.json) 放入该文件夹。
3. 在 Obsidian 设置 → 社区插件 → 已安装，启用 **Local Image Manager**。

---

## ⚙️ 配置说明

在插件设置中可调整：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| **基础存储文件夹** | 图片将保存在此文件夹下，后接笔记的目录和文件名。 | `Assets/Image` |
| **标题层级最大深度** | 生成文件名时最多使用几级标题序号（1-6）。超出部分将被截断。 | `6` |
| **粘贴图片行为** | `总是保存`：自动保存；`每次询问`：弹出确认窗口。 | `总是保存` |
| **默认链接格式** | 选择插入图片时使用的链接格式（Wiki 或 Markdown）。 | `Wiki` |

## 🚀 使用方法

### 基本流程
1. 打开一篇带有标题的笔记（例如 `DL/ANN.md`，包含 `#`、`##` 等标题）。
2. 将光标放在某个标题下方（该位置决定了图片的命名层级）。
3. 粘贴图片（`Ctrl+V` / `Cmd+V`）或从外部拖拽图片到编辑器。
4. 插件会自动：
   - 计算当前标题的编号（如 `1.2.3`）。
   - 在该层级下递增计数器（如 `1.2.3-1.png`）。
   - 将图片保存到 `Assets/Image/DL/ANN/` 目录。
   - 在光标处插入 `![[相对路径]]`。

### 后备场景
如果光标不在任何标题下，插件会：
- 弹出提示：“当前光标位置无有效标题，建议在标题层级下粘贴图片以获得有序命名。”
- 使用 `笔记名-时间戳.扩展名` 作为文件名（例如 `ANN-20260703T120000.png`）。
- 图片仍按笔记路径存放，保证管理一致。

### 右键菜单
- **编辑器（源码模式）下**：将光标放在图片链接（`![[...]]` 或 `![]()`）上，右键可执行：
  - `重新整理本笔记图片序号`
  - `删除此图片（文件与链接）`
- **阅读/实时预览模式**：由于 Obsidian 内部机制的限制，插件不会接管图片的右键菜单。您仍可通过命令面板（`Ctrl+P`）或自定义快捷键执行上述功能。
